# Contributing to Free Tech Resources

First off — thank you! 🎉 This list grows by people like you sharing great resources.

## How to Contribute

### Option 1: Submit an Issue (Easiest)
Don't want to mess with code? Just [open an issue](../../issues/new) and paste:
- Resource name
- URL
- What category it fits
- Why it's worth adding

We'll handle the rest.

### Option 2: Submit a Pull Request

1. **Fork** this repository (click the Fork button top-right)
2. **Clone** your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/free-tech-resources.git
   ```
3. **Create a branch:**
   ```bash
   git checkout -b add-my-resource
   ```
4. **Add your resource** to the correct section in `README.md`
5. **Commit your change:**
   ```bash
   git add .
   git commit -m "Add [Resource Name] to [Category]"
   ```
6. **Push** to your fork:
   ```bash
   git push origin add-my-resource
   ```
7. **Open a Pull Request** — describe the resource and why it belongs

## Resource Requirements

To keep this list high quality, every resource must be:

| Requirement | Details |
|---|---|
| ✅ Free | No credit cards, no paywalls, no "free trial only" |
| ✅ Useful | Solves a real learning need |
| ✅ Active | Updated in the last 2 years OR a timeless classic |
| ✅ English | For now, the list is English-only |
| ✅ Safe | No spam, phishing, or low-quality content |

---

## Formatting Guidelines

Use this format when adding to a table:

```markdown
| [Resource Name](https://url.com) | Short description | Level/Format |
```

- Keep descriptions short — one sentence max
- Link text = the resource's actual name
- Add to the most relevant category
- Maintain alphabetical order within a category if possible

---

## What NOT to Submit

- Paid courses or tools (even "freemium" with very limited free tiers)
- Duplicates of resources already listed
- Your own product unless it's genuinely well-known and widely used
- Dead links or abandoned projects
